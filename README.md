# Sacramento Church Website

This was a professional, production-grade web application built for Sacramento Wesleyan Methodist Church. Developed as a transition from Flask to Django, this project served as the church's digital front door for over a year. It was designed to handle dynamic scheduling, multimedia distribution, and community outreach.

## Tech Stack
- Backend: Python (Django 4.1.7)
- Frontend: HTML5, CSS3, JavaScript (Es6+)
- Database: SQLite (Development) / PostgreSQL (Production)
- Styling: Bootstrap 5.3.0 for layout, supplemented by custom CSS.
- Integration: django-recaptcha for secure communication.

## Key Features
- Dynamic Sermon Archive: A robust media management system for uploading and streaming sermon audio, allowing the congregation to access preaching remotely.
- Interactive Articles of Faith: A custom-built interface for navigating complex theological documents. I developed a JavaScript “scroll-to” system that links section headers to an easy-to-use navigation index, creating a smoother experience for reading and moving through long-form content.
- Production Deployment: Managed the site in a live environment for a one-year trial period, handling real-world updates and maintenance.
- Security & Anti-Spam: Implemented Google reCAPTCHA on all public-facing forms to prevent bot-driven spam, alongside Django’s native CSRF and XSS protections.

## What I Learned
- Framework Adaptability: Successfully transitioned from Flask’s lightweight approach to Django’s full-featured framework, gaining hands-on experience with the ORM, admin interface, and powerful templating system.
- Real-World Stakeholder Experience: Successfully pitched, developed, and deployed a technical solution for a non-technical organization. This taught me how to communicate technical benefits to stakeholders and manage a project through its entire lifecycle.
- Frontend Problem Solving: Spent significant time perfecting the JavaScript for the "Articles of Faith" page, which deepened my understanding of DOM manipulation and smooth-scrolling behavior.

## Status
 - Project completed; organization decided to close the website due to no fault of the website.
